## fanqie-novel-skill 审计系统集成 (22:00)

### 架构升级
用户提出新的系统架构（事件驱动 + 审计驱动 + 状态锁定）：
- Chapter Writer（生成章节）
  ↓
- Event Flow System（剧情事件生成/执行）
  ↓
- 🔥 fanqie-novel-skill（27维审计 + AI味检测 + 平台规则）← 质量闸门
  ↓
- World State System（更新世界状态）
  ↓
- Story Memory（记录章节摘要）

### 实现内容
1. **创建 fanqie-novel-skill 后端模块**（src/server/audit/fanqieSkill.ts）
   - AI味检测器（套路词/空洞表达/排比句）
   - 逻辑审计（27维：时间线/设定一致性/行为合理性）
   - 平台规则检查（300字主角/1000字钩子）
   - 伏笔追踪（Truth Files）
   - 输出：{ status, score, issues, suggestions }

2. **更新章节生成 API**（src/app/api/generate-chapter/route.ts）
   - 生成章节 → 审计 → 根据结果决定是否更新世界状态
   - 返回审计结果给前端

3. **创建审计面板前端组件**（src/components/chapter/AuditPanel.tsx）
   - 显示审计结果（AI味/逻辑问题/平台风险/伏笔遗漏）
   - 展示修改建议
   - 用户操作：确认通过/请求修改/拒绝

4. **更新 Chapter View 页面**（集成审计流程）
   - 新流程：生成 → 审计 → 确认 → 保存
   - 添加审计报告 Tab
   - 根据审计结果更新世界状态
